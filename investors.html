Token Economics
How LUSH Works

A controlled inflation model with deflationary burns. Users earn LUSH through verified engagement and spend it on virtual gifts that permanently destroy the tokens.

Mint & Burn Flow

↑

Mint — Earn Events

GPS check-in (+10), first nightly check-in (+25), streak bonuses (+50–1,000), swarm activity (+5–15), challenges (+25–75)

↔

Hold — User Wallet

Privy embedded wallet (non-custodial, MPC key sharding). Users see "Lush Coins: 47" — never wallet addresses or transaction hashes

↓

Burn — Virtual Gifts

21 paid gift items (10–1,000 LUSH). Tokens are permanently destroyed on purchase. Future: cosmetics, promoted events, venue drink redemptions

Technical Stack
Built for Production, Not Demo Day

A battle-tested stack designed for real-world nightlife with instant blockchain fallback. Every component is live and deployed.

SOLANA

SPL Token on Solana

~400ms finality, ~$0.003/tx. Standard SPL token program — audited, battle-tested. No custom smart contracts, no attack surface.

PRIVY

Embedded Wallets

MPC key sharding via Privy (acquired by Stripe). Auto-created on signup, invisible to users. Non-custodial, recoverable, no seed phrases.

RADAR

GPS Geofencing

Radar.io validates every venue check-in. No fake engagement — LUSH is only minted when a user is physically verified at a registered venue.

SUPABASE

Edge Functions

25+ deployed Edge Functions. Server-side mint authority — the client can never mint tokens. All minting requires verified earn events with anti-fraud checks.

FEATURE FLAG

Instant Fallback

VITE_CRYPTO_ENABLED toggles blockchain on/off. App works identically with database-only accounting. Zero user impact during Solana outages.

CAPACITOR

Cross-Platform

PWA + native iOS + Android via Capacitor. Single React/TypeScript codebase. App Store and Google Play submission-ready.
